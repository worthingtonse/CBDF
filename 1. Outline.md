# Outline 
This is the outline of the entire Qmail CBDF file.
Each part of the qmail file can be seperate files. This allows the same Style section to be used by many pages (in the future) so that the code only needs to be downloaded
once. 

Here we are only showing the seperators of the different parts of the qmail document

## Declaring the constants: 
The command characters are non-printable ASCII characters theat are used command. 
```c
typedef enum{
    FILE_SEPERATOR = 28, //  Seperates meta, styles, text and logic
    GROUP_SEPERATOR = 29,_ // Seperates subparts of each group
    RECORD_SEPERATOR = 30, // Seperates the sub parts of the sub parts
    UNIT_SEPERATOR = 31  // Etc. 
} command_type_t
```
## Command Keys
Here is how the files are organized. Even if there are no content between the seperators, the seperators are a fixed part of the file. 

```C
      Meta_Data 
A1. FILE_SEPERATOR // Devide between Meta_Data and Style_Code
    B1. GROUP_SEPERATOR:
      Page_style (includes how many containers, other stuff)
    B2. GROUP_SEPERATOR -- Between background and containers
        C1. RECORD_SEPERATOR
            Container_Style[0]
        C2. RECORD_SEPERATOR
            Container_Style[1]
        C3. RECORD_SEPERATOR
            Container_Style[2]
    B3. GROUP_SEPERATOR -- Between contaner Styles and text styles
        C1. RECORD_SEPERATOR
            Text_Style[0]
        C1. RECORD_SEPERATOR
            Text_Style[1]
      Text_Styles
    B4. GROUP_SEPERATOR
      Container_Styles
A2. FILE_SEPERATOR // Devide between Style and Text
    B1. GROUP_SEPERATOR
      Background_Layer 
    B2. GROUP_SEPERATOR
        C1. RECORD_SEPERATOR
            Container_Layer[0]
        C2. RECORD_SEPERATOR (Optional. This is an array of Containers)
            Container_Layer[1] 
    B3. GROUP_SEPERATOR   
        C1. RECORD_SEPERATOR        
          Container_Content[0]
        C2. RECORD_SEPERATOR        
          Container_Content[1]
        C3. RECORD_SEPERATOR        
          Container_Content[2]
        C4. RECORD_SEPERATOR        
          Container_Content[3]
A3. FILE_SEPERATOR // Devide between Text and Logic
    //This is the end of phase three. Logic will be added later.
```
