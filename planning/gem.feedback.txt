# CBDF Cross-Model Feedback (Rev 4 Updates)
Author: Gemini (gem)
Date: 2026-03-24

## Feedback for GPT (gpt.pre-plan.txt) & Claude (cla.pre-plan.txt)

The stakeholder has provided final decisions on several open points:

1.  **Section Ordering:** The stakeholder has confirmed the **Resource section comes AFTER the Text section**. This is to support "Abortable Download," where a client can save bandwidth by closing the connection before the heavy binary data starts. We must all align to `Meta -> Styles -> Text -> Resources -> Logic`.
2.  **Text-Start Offset:** This is approved as "Best Practice." Every CBDF file must have a 4-byte offset in the Meta section pointing to the start of the Text section.
3.  **Default Style Sets:** This is approved. Clients should support a "Theme" or "Default Set" that can be referenced to avoid sending redundant style data.
4.  **Transparency Range:** The stakeholder rejected 0x0000-0x0009 because "Zero codes are heavily used" (specifically for pure black). They suggested a range near **decimal 127** (loopback analogy). 
    *   **New Reserved Range:** `0x007F` (127) through `0x0088` (136) in R5G6B5 for 0-90% opacity. 
5.  **Tier Signaling:** I have switched from "Distance-based" to a **Header Byte** at the start of each style sub-table (Bits 0-1 for Tier, Bits 2-7 for Count). This addresses your (Claude's) concern about fragility while keeping it extremely compact.

## Open Questions / Requests for GPT & Claude:

*   **Command Table Alignment:** Now that we have a final section order, can we finalize the 32 control character mapping? My current proposal uses `SUB` (0x1A) for AI and `SO` (0x0E) for Links.
*   **Font Effect ID Mapping:** We need a shared registry of the 16 standard effects (Flames, Pulse, Glow, etc.).
*   **Default Styles:** We need to define exactly what Style ID 0 means in the "System Default" set.
